# EEG Insights Agent (EEGIA)

## 📋 Описание

Этот проект представляет собой исследовательскую среду в Jupyter Notebook для сравнительного анализа различных методов классификации сигналов ЭЭГ.

## 🚀 Основные возможности:

### 📥 Поддержка датасетов
Проект автоматизирует загрузку и предобработку данных из трех фундаментальных источников:
* **EEG BCI**: Классификация моторного воображения (движение рук/расслабление).
* **MNE Sample**: Анализ первичных сенсорных ответов (аудиальные vs визуальные стимулы).
* **ERP Core**: Исследование когнитивного контроля и нагрузки (задача Фланкера).

---

### 🧠 Алгоритмы обработки (Pipeline)
1.  **ICA (Independent Component Analysis)**: Автоматический поиск и удаление артефактов моргания глаз.
2.  **Preprocessing**: Ресемплинг до 160 Гц, полосовая фильтрация (1–40 Гц) и Average Reference.
3.  **Epoching**: Нарезка сигнала на эпохи с коррекцией базовой линии (Baseline: -200мс до стимула).

---

## 🔬 Сравниваемые пайплайны (Модели)
Проект сопоставляет четыре различных подхода к классификации. Каждый пайплайн включает в себя свой метод извлечения признаков:

| Пайплайн | Экстрактор признаков (Feature Extraction) | Классификатор |
| :--- | :--- | :--- |
| **Hybrid** | **Custom Hybrid**: Спектральная мощность (Welch PSD) + ROCKET (случайные свертки). | **Random Forest** (с балансировкой весов) |
| **Rocket** | **Sktime Rocket**: Преобразование сигнала через 2000+ случайных ядер. | **RocketClassifier** |
| **CSP_LDA** | **CSP**: Общие пространственные паттерны (максимизация дисперсии между классами). | **LDA** (Линейный дискриминант) |
| **CSP_Forest**| **CSP**: Пространственная фильтрация. | **Random Forest** |

---

## 📊 Валидация и Оптимизация
Для получения объективных результатов в проекте реализованы:

* **GridSearchCV**: Автоматический перебор гиперпараметров (количество ядер ROCKET, глубина леса, компоненты CSP).
* **Stratified K-Fold**: 5-кратная кросс-валидация с сохранением пропорций классов.
* **Метрики**:
    * `Accuracy`: Общая точность классификации.
    * `F1-Macro`: Качество классификации, не зависящее от баланса классов.
    * `Time`: Время обучения и оптимизации каждой модели.

---

## 📈 Визуализация
### 1. Confusion Matrix (Матрица ошибок)
Для каждой комбинации «Датасет + Модель» строится тепловая карта (Heatmap), которая позволяет:
* Определить «конфликтные» классы.
* Оценить смещение модели в сторону одного из классов.

### 2. Сводная таблица результатов
Итоговый `DataFrame` со всеми метриками и лучшими найденными параметрами (`Best Params`) для финального сравнения эффективности алгоритмов.
